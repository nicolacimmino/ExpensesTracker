What is this?

This is not a live service but an application developed to try and familiarize with a modern Web development stack. You can find all source code in Github. This is just the Web front-end. Behind the scenes there is a ReSTful API that takes care of the data and security and there is also a companion Android application you can use to log expenses while on the go.
 
 

Web Application

The Web application is the one you are looking at right now. It's an SPA (Single Page Application) built in Angular.js and made nearly pretty with the help of Bootstrap CSS. The pages and the resources are served by a Node.js server, but just out of convenience, there is really no processing or access to data on the server side, we only serve static HTML, CSS and JavaScript, all processing is done on the client in Agular.

API

The Web Application interacts with the actual user data trough a HTTP ReSTful API. This is realized with a Node.js server using Express and Monk to access a MongoDB. The API allows the applications to authenticate a user and provides CRUD methods to manipulate single expenses belonging to one user. The same API is used by the Web Application and the Android Application.
 
 

Android

The Android application allows to easily report expenses while on the go. Its main interface is designed to be usable also when walking and using the phone single-handed. The application itself dosen't offer much more than the ability to report new expenses. It's anyhow an interesting reference design for an Android application as it shows how to create Content Providers and Sync Adapters in order to perform background data sync in the most battery efficient way. It also demonstrates how to implement an Account Authenticator so that the user account is managed properly by Android. The application makes use of a SQLite database to store locally the data until it's sent over to the server trough the API.
